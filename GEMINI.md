# Requisitos a Implementar (Sentinel SRaC) - Roadmap Inspirado no SD Elements

Lista de funcionalidades para expansão da plataforma Sentinel SRaC, visando maior maturidade e integração com o fluxo de desenvolvimento.

- [x] **Mapeamento Multicontrol (Compliance)**
    - Exibir como um único requisito atende a múltiplas normas (OWASP ASVS, PCI-DSS, NIST 800-53, LGPD/GDPR).
- [x] **Status de Implementação e Notas de Verificação**
    - Rastrear o progresso (Pendente, Implementado, N/A).
    - Permitir que o desenvolvedor anexe notas sobre a solução adotada.
- [ ] **Integração com Issue Trackers (Jira/GitHub Issues)**
    - Botão para exportar requisitos diretamente como tarefas em boards de desenvolvimento.
- [x] **Instruções de Verificação (Como testar)**
    - Adicionar seções de "Como Testar" para cada requisito, incluindo snippets de testes unitários ou comandos de ferramentas DAST.
- [x] **Priorização Baseada em Risco**
    - Atribuir níveis de Severidade (Crítico, Alto, Médio, Baixo).
    - Cálculo de Score de Risco inicial para o projeto.
- [ ] **Questionário Dinâmico (Lógica Condicional)**
    - Adaptar perguntas conforme as respostas anteriores (ex: se usar SQL, perguntar sobre sanitização de queries).
- [ ] **Biblioteca de Treinamento (Just-in-Time Learning)**
    - Links para OWASP Cheat Sheets ou vídeos curtos explicando a vulnerabilidade e a correção.
